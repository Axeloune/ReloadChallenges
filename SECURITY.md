# Security Policy

## Versions supportées

Le projet vise la branche principale et la version Paper indiquée dans `pom.xml`.

## Signaler une vulnérabilité

Pour une faille de sécurité, évite de publier un rapport exploitable dans une issue publique.

Ouvre plutôt une discussion privée avec le mainteneur du dépôt, ou contacte le propriétaire du projet via le canal indiqué sur GitHub si disponible.

Inclure si possible :

- version du plugin ;
- version Paper ;
- étapes de reproduction ;
- impact estimé ;
- logs utiles sans données sensibles.
